# 今天吃什么 - 微信小程序

解决每日饮食选择困难的AI问答式微信小程序。

## 项目目标
- **AI问答流式交互**：通过自然对话收集用户偏好
- **个性化推荐**：基于用户特征（喜好、健康、忌口等）智能推荐
- **用户数据存储**：建立个人特征库和知识库
- **微信小程序平台**：完整的移动端体验

## 技术栈
- **前端**：微信小程序 (WXML + WXSS + JavaScript)
- **后端**：微信云函数（Node.js）
- **AI交互**：流式对话引擎
- **数据存储**：微信云开发（云数据库 + 云存储）
- **部署**：微信小程序平台

## 核心功能模块
1. **对话引擎**：处理用户输入，生成流式响应
2. **用户画像**：收集和管理用户特征（喜好、忌口、健康状况等）
3. **推荐算法**：基于用户画像的个性化美食推荐
4. **菜谱数据库**：结构化美食数据
5. **历史记录**：保存用户的推荐历史和反馈

## 微信云函数详解

### 什么是微信云函数？

微信云函数是微信云开发提供的一种**Serverless 云服务**，允许开发者在云端运行代码而无需管理服务器。云函数具有以下特点：

- **无服务器架构**：无需购买、配置和维护服务器
- **按需计费**：只在函数被调用时产生费用
- **自动扩缩容**：根据请求量自动调整资源
- **集成云开发**：与云数据库、云存储无缝集成
- **安全可靠**：微信官方托管，具备完善的安全机制

### 微信云函数与小程序的关系

微信云函数与小程序之间是**前后端分离**的关系：

```
小程序（前端） ↔ 云函数（后端） ↔ 云数据库/云存储
```

**具体关系说明：**

1. **数据处理**：小程序负责用户界面和交互，云函数负责业务逻辑和数据处理
2. **安全边界**：敏感操作（如数据库写入、用户认证）在云函数中执行，避免前端直接操作数据库
3. **权限控制**：云函数运行在服务端环境，拥有更高的权限，可以执行前端无法完成的操作
4. **性能优化**：复杂的计算和数据处理在云端完成，减轻小程序端的负担

### 通信机制

小程序与云函数通过 **`wx.cloud.callFunction()`** API 进行通信：

#### 调用方式
```javascript
// 小程序端调用云函数
wx.cloud.callFunction({
  name: '函数名称',        // 云函数文件夹名称
  data: {               // 传递给云函数的参数
    key: 'value'
  },
  success: res => {
    // 处理云函数返回结果
    console.log(res.result);
  },
  fail: err => {
    // 处理调用失败
    console.error(err);
  }
});
```

#### 云函数响应
```javascript
// 云函数端
exports.main = async (event, context) => {
  // event 包含小程序传递的 data 参数
  // context 包含微信上下文信息（如 openid）
  
  // 执行业务逻辑
  const result = await doSomething(event.data);
  
  // 返回结果给小程序
  return {
    success: true,
    data: result
  };
};
```

#### 数据流向
1. **请求**：小程序 → 云函数（通过 `data` 参数）
2. **上下文**：微信自动注入用户信息（openid、appid 等）
3. **响应**：云函数 → 小程序（通过 `return` 语句）

### 本项目中的云函数

本项目使用了两个核心云函数：

#### 1. `checkUserExists` - 用户存在性检查
- **功能**：检查当前用户是否已存在
- **输入**：无（自动获取用户 openid）
- **输出**：用户存在状态和偏好设置
- **使用场景**：小程序启动时检查用户登录状态

#### 2. `saveUserInfo` - 保存用户信息
- **功能**：保存或更新用户信息和偏好设置
- **输入**：用户基本信息和偏好设置
- **输出**：操作成功/失败状态
- **使用场景**：新用户注册或偏好更新时

### 发布和维护流程

#### 开发环境设置
1. **安装微信开发者工具**
   - 下载并安装最新版本的微信开发者工具
   - 登录开发者账号

2. **项目导入**
   - 在开发者工具中导入项目
   - 选择项目目录：`/projects/today-what-to-eat-wechat`
   - 配置 AppID（测试可使用测试号）

3. **云环境配置**
   - 在开发者工具中开通云开发
   - 创建开发环境（建议命名为 `dev`）
   - 记录环境 ID 用于本地配置

#### 云函数部署

##### 首次部署
1. **上传云函数**
   - 在开发者工具中右键 `cloudfunctions` 文件夹
   - 选择 **"上传并部署：所有文件"**
   - 等待部署完成

2. **验证部署**
   - 在云开发控制台查看云函数列表
   - 测试云函数是否正常运行

##### 后续更新
1. **修改云函数代码**
   - 在本地修改云函数文件
   - 确保代码符合 Node.js 规范

2. **增量部署**
   - 右键修改的云函数文件夹
   - 选择 **"上传并部署：云端安装依赖"**
   - 或者使用命令行工具批量部署

#### 小程序发布

##### 测试阶段
1. **本地调试**
   - 在开发者工具中编译运行
   - 使用真机调试功能测试实际效果

2. **体验版发布**
   - 在开发者工具中点击 **"预览"**
   - 生成体验版二维码供测试

##### 正式发布
1. **代码上传**
   - 在开发者工具中点击 **"上传"**
   - 填写版本信息和项目备注

2. **审核提交**
   - 登录微信公众平台
   - 在版本管理中提交审核
   - 等待微信团队审核（通常 1-7 天）

3. **上线发布**
   - 审核通过后，点击 **"发布"**
   - 小程序正式对用户可见

#### 维护最佳实践

##### 版本管理
- **Git 分支策略**：
  - `main`：生产环境代码
  - `develop`：开发环境代码
  - `feature/*`：功能分支
- **云函数版本**：每次重大更新前备份当前版本

##### 监控和日志
- **云开发监控**：定期查看云函数调用次数、错误率
- **日志分析**：通过云开发控制台查看函数执行日志
- **性能优化**：监控函数执行时间和内存使用

##### 安全考虑
- **权限最小化**：云函数只授予必要的数据库权限
- **输入验证**：对所有用户输入进行严格验证
- **敏感信息**：不要在代码中硬编码敏感信息

##### 成本控制
- **调用优化**：减少不必要的云函数调用
- **资源监控**：关注云开发资源使用情况
- **免费额度**：充分利用微信云开发的免费额度

### 故障排查

#### 常见问题
1. **云函数未部署**
   - 错误：`cloud function not found`
   - 解决：重新上传部署云函数

2. **权限不足**
   - 错误：`permission denied`
   - 解决：检查云函数和数据库的权限设置

3. **网络超时**
   - 错误：`timeout`
   - 解决：优化云函数逻辑，减少执行时间

#### 调试技巧
- **本地调试**：使用开发者工具的云函数调试功能
- **日志输出**：在云函数中添加详细的 `console.log`
- **单元测试**：为云函数编写独立的测试用例
